This is the most standard approach. By default, the Lucid Tabs plugin will search within the element you call it on for anchors with href values pointing to IDs on the page. These anchors become the tabs, and the elements with the matched IDs become the “pages” that are shown or hidden when the selected tab changes.

This default setup has the added benefit that users without JavaScript enabled will still have working in-page navigation.

If you take a peak at the source, you’ll notice that the markup for the accordion is a definition list. To get Lucid Tabs to recognize the DT elements as the tabs and the DD elements as the pages, you need to override the default options locateTabs and locatePages with your own functions that return jQuery references to the desired elements.
